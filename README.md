# Continental Intelligence Platform

An FT/Economist-styled intelligence dashboard covering economic, human
development, health, environment, and governance indicators across 36
countries — live from the World Bank and WHO, with AI-written executive
briefings. Built by [Inscend Consulting Limited](https://www.inscend.org/).

**Live site:** https://fadl462.github.io/continental-intelligence-platform/

## Architecture — no server, no database, just GitHub

| Layer | What it is | Where |
|---|---|---|
| Hosting | Static site | GitHub Pages, serving `index.html` from `main` |
| Live data | Client-side `fetch()` straight from public APIs | `api.worldbank.org`, `ghoapi.azureedge.net` — called directly from the browser, no key needed |
| AI briefings | Pre-generated executive summaries | `data/briefings.json`, committed to the repo |
| Compute | Scheduled job that (re)generates briefings | GitHub Actions — `.github/workflows/generate-briefings.yml` |

There is no Render/Supabase/FastAPI anywhere in this version. The dashboard's
charts, KPIs, and rankings work purely client-side (no setup needed at all —
that part is already live). The **only** piece that genuinely needs
server-side compute is calling the Anthropic API for AI briefings (an API
key can't safely live in public HTML), and GitHub Actions handles that on a
schedule, writing the result back into the repo as a plain JSON file that
the page reads like any other static asset.

## Continental AI

The bottom-right assistant has two tiers:

1. **Live data Q&A** (instant, works right now) — a rules-based parser that
   reads your question, pulls the relevant live World Bank/WHO figures, and
   answers. Try: *"Ghana's GDP per capita"*, *"compare Nigeria and Kenya on
   inflation"*, *"top 5 in Africa for life expectancy"*, *"show me the
   governance tab"*.
2. **Executive briefings** (pre-generated, refreshed weekly) — ask
   *"briefing on Ghana"* and it returns the cached Claude-written summary
   from `data/briefings.json`, generated by the Action below.

## Setting up the AI briefings

1. **Add a repo secret**: Settings → Secrets and variables → Actions → New
   repository secret → name it `ANTHROPIC_API_KEY`, value = your Anthropic
   API key.
2. **Enable write permissions for Actions**: Settings → Actions → General →
   under "Workflow permissions", select **Read and write permissions**, then
   Save. (Needed so the job can commit `data/briefings.json` back to the repo.)
3. **Run it once manually**: Actions tab → "Generate AI Briefings" →
   **Run workflow**. It'll fetch each country's latest indicators, ask
   Claude for a short executive summary, and push `data/briefings.json`.
4. After that, it runs automatically every Monday at 06:00 UTC
   (`cron: "0 6 * * 1"` in the workflow file) — no further action needed.

To limit scope (fewer countries, faster/cheaper runs), set an optional
`BRIEFING_COUNTRIES` env var in the workflow step, e.g. `GHA,NGA,KEN,ZAF`.

## Local development

```bash
# Frontend — just open it, no build step
open index.html

# Briefing script — run locally against your own key
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python scripts/generate_briefings.py
```

## Repo structure

```
index.html                          the entire frontend — dashboard + Continental AI
assets/                              Inscend logo files
data/briefings.json                  AI-generated executive briefings (auto-committed)
scripts/
  fetch_worldbank.py                 World Bank API client
  fetch_who.py                       WHO GHO API client
  config.py                          country list + which indicators feed each briefing
  generate_briefings.py              the script the Action runs
.github/workflows/generate-briefings.yml   weekly scheduled job
requirements.txt                     requests + anthropic (that's it)
```

## Extending it

- **More countries**: add to `COUNTRIES` in both `scripts/config.py` and the
  `CONTINENTS` object in `index.html`.
- **More indicators in the briefing prompt**: add to `BRIEFING_INDICATORS`
  in `scripts/config.py`.
- **Faster briefing refresh**: change the cron schedule in
  `.github/workflows/generate-briefings.yml`.
